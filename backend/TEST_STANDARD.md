# 智能助手测试正确性标准

## 一、测试目标

确保智能助手的搜索功能和AI能力达到主流软件AI搜索的水平，具体包括：
- 意图识别准确率高
- 搜索结果相关性强
- 帖子展示格式统一（可点击）
- 对话上下文感知
- 错误处理合理

## 二、测试正确性标准

### 2.1 意图识别正确性标准

| 标准编号 | 标准描述 | 测试验证方法 |
| :--- | :--- | :--- |
| IR-001 | 问候语（如"你好"、"您好"）应被正确识别 | 发送问候语，验证返回类型为"text"且包含欢迎信息 |
| IR-002 | 感谢语（如"谢谢"、"感谢"）应被正确识别 | 发送感谢语，验证返回礼貌回应 |
| IR-003 | 帮助请求（如"能做什么"、"怎么用"）应被正确识别 | 发送帮助请求，验证返回功能列表 |
| IR-004 | 技术关键词（如"云计算"、"Vue"）应触发搜索意图 | 发送技术关键词，验证返回类型为"posts" |
| IR-005 | "总结"类请求不应被误判为搜索请求 | 发送"总结一下该帖子"，验证不走搜索路径 |
| IR-006 | "帮我找XX"不应被误判为帮助请求 | 发送"帮我找Vue帖子"，验证返回搜索结果 |
| IR-007 | 需要登录的意图（如"我的帖子"）需检查用户状态 | 未登录时发送"我的帖子"，验证返回登录提示 |

### 2.2 搜索功能正确性标准

| 标准编号 | 标准描述 | 测试验证方法 |
| :--- | :--- | :--- |
| SR-001 | 搜索结果必须返回统一的"posts"格式 | 检查返回数据结构包含 id, title, category, viewCount, createdAt |
| SR-002 | 搜索结果必须按匹配度排序 | 使用多关键词搜索，验证相关度高的帖子排在前面 |
| SR-003 | 搜索结果应包含匹配度分数 | 检查返回数据包含 matchScore 字段 |
| SR-004 | 标题匹配权重应高于内容匹配 | 相同关键词，标题匹配的帖子应排在内容匹配前面 |
| SR-005 | 热门帖子应获得额外权重 | 相同匹配度下，浏览量高的帖子应排在前面 |
| SR-006 | 最新帖子应获得时间优势 | 相同匹配度下，近期发布的帖子应排在前面 |
| SR-007 | 未找到结果时应返回推荐帖子 | 搜索不存在的关键词，验证返回最新帖子推荐 |

### 2.3 帖子格式统一标准

| 标准编号 | 标准描述 | 测试验证方法 |
| :--- | :--- | :--- |
| PF-001 | 所有搜索路径返回的帖子格式必须一致 | 对比本地搜索和AI路径返回的数据结构 |
| PF-002 | 分类字段必须统一为中文 | 验证category字段值为"技术"、"新闻"、"产品"、"其他" |
| PF-003 | 时间格式必须统一 | 验证createdAt字段为"YYYY/MM/DD"格式 |
| PF-004 | 返回类型必须明确 | 验证type字段为"posts"或"text" |

### 2.4 AI路径正确性标准

| 标准编号 | 标准描述 | 测试验证方法 |
| :--- | :--- | :--- |
| AI-001 | 纯粹总结请求应走AI路径 | 发送"总结一下该帖子"，验证返回AI总结而非帖子列表 |
| AI-002 | 无技术关键词的问题应走AI路径 | 发送一般性问题，验证返回AI回答 |
| AI-003 | AI回复中的帖子信息应自动转换为posts格式 | AI回复包含帖子时，验证自动转换为可点击格式 |
| AI-004 | 对话历史应被正确使用 | 连续对话时，验证AI理解上下文 |
| AI-005 | AI服务降级机制应正常工作 | 模拟AI服务故障，验证降级到本地搜索 |

### 2.5 错误处理正确性标准

| 标准编号 | 标准描述 | 测试验证方法 |
| :--- | :--- | :--- |
| EH-001 | 空问题应返回错误提示 | 发送空字符串，验证返回400错误 |
| EH-002 | 无效输入应返回友好提示 | 发送乱码或无意义字符，验证返回合理提示 |
| EH-003 | 未登录用户请求个人数据应提示登录 | 未登录时请求"我的帖子"，验证返回登录提示 |
| EH-004 | 服务异常应返回友好提示 | 模拟数据库故障，验证返回服务不可用提示 |

### 4.3 整体通过标准
- 功能测试通过率 ≥ 95%
- 性能测试全部达标

## 五、测试执行流程

1. **启动服务器**: 确保后端服务运行在 http://localhost:3000
2. **运行测试**: 执行 `node test_comprehensive.mjs`
3. **检查结果**: 查看测试报告，确认各测试用例状态
4. **定位问题**: 对失败的测试用例进行调试和修复
5. **回归测试**: 修复后重新运行测试，确保问题已解决

运行测试前请确认：
- [ ] 服务器已启动并运行在正确端口

- [ ] 数据库中有足够的测试数据（至少10篇帖子）
- [ ] 测试脚本使用正确的API端点
